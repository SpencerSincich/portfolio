Software development tools



A software development tool is a type of software used by a developer to create new and improve existing software. For this article, software is defined as a set of electronic instructions that tell the hardware (physical components of the computer) what tasks to perform.



Source code editors



A source code editor is software used by a developer to type text instructions in a high-level programming language. This is a source language that is easiest for a developer to describe instructions in, such as C, C++, Java, PHP, Python, Visual Basic, etc.


The instructions the developer types in the source code editor are like the first steps of a cake recipe being precisely followed. Everything is prepared before the step of putting the ingredients in the oven.



Compilers and interpreters



Both compilers and interpreters are software development tools used to translate the instructions typed in the source code editor. The instructions are translated from a high-level language to a target language that is, or is closer to, machine code. This is a language that hardware understands and uses to perform tasks.



A compiler translates the entire set of instructions at once, before runtime. So the pre-translated instructions are ready for use in the target language.


An interpreter translates the specified subset of instructions at runtime. So the subset of instructions are translated to the target language only when required.



Reasons for using compilers and/or interpreters



For most practical purposes, each programming language is usually considered either a compiled or an interpreted language. However, in some scenarios, developers can use a compiler and/or interpreter to translate code from the same source language


What remains the same is: For the instructions to work on hardware, the source code must eventually be translated to machine code.



    The developer's decision to use a compiler and/or interpreter is based on weighing the advantages and disadvantages of each process. The decisive criteria is usually:
  • Portability - Which machines will the software work on?
  • Performance - What will optimize the speed of the software?

Compiled instructions have low portability, and are high performance.

For example: A developer types source code in C that is compiled into machine code.


The disadvantage of compiling instructions is the compiled code has low portability. This means the instructions can work on fewer types of hardware.


The advantage of compiling instructions is the compiled code is high performance. This means the instructions work faster.


Interpreted instructions have high portability, and are low performance.

For example: A developer types source code in PHP that is interpreted into machine code.


The advantage of interpreting instructions is the source code has high portability. This means the same PHP code can work on many types of hardware — with the help of interpreters.


The disadvantage of interpreting instructions is the interpreted code is low performance. This means the instructions work slower, because of the required interpretation step. However, the interpreted code is still fast enough for most purposes.



A more complex process occurs when developers type source code in Java, and use both a compiler and an interpreter to translate the code.


First the Java source code is compiled into an intermediate language that is a type of machine code, called "bytecode". This code is then executed by an interpreter called the JVM (Java Virtual Machine).


By using both a compiler and an interpreter to translate the Java source code, developers can gain the advantages of using each. They can also minimize the disadvantages.



Conclusion



The purpose of software development tools is to provide developers with what's necessary to create new and improve existing software. While 3 software development tools are described here, there are many others, such as debuggers, profilers, linkers, etc. For some programming purposes, there is a trend towards the use of IDEs (integrated development environments). IDEs include several software development tools in one software product.
